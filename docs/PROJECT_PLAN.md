## Goal

Build a searchable, SEO- and AEO-optimized (Answer Engine Optimization) directory of medical terms centered on:

- **Thyroid** — Hashimoto's, hypothyroidism, Graves', hyperthyroidism, nodules, cancer, thyroiditis
- **Hormone health** — sex hormones, adrenal, perimenopause/menopause, PCOS
- **Endocrinology** — GLP-1, insulin resistance, diabetes, metabolic syndrome
- **Weight loss** — GLP-1 medications, metabolic health

Each entry covers: conditions, diagnoses, concerns, symptoms, medications, supplements, labs/diagnostics, procedures, and key concepts used by MTC clinicians and patients.

## Objectives

1. ~200 authoritative term entries, each with interlinking to related terms.
2. Schema.org-structured definitions that rank well in search and get surfaced by AI answer engines (ChatGPT, Perplexity, Google AI Overviews).
3. Webflow CMS collection drives a Directory landing page, a searchable A–Z list, and a record detail page.
4. Design matches MTC brand (existing variable collection + style guide).
5. Directory link added to primary nav.

## CMS Collection Schema

| Field Name | Type | Required | Purpose |
|---|---|---|---|
| Name | PlainText (auto) | ✅ | Term display name |
| Slug | PlainText (auto) | ✅ | URL path |
| Category | Option | ✅ | Condition / Symptom / Medication / Supplement / Lab / Procedure / Concept |
| First Letter | Option | ✅ | A–Z (for alphabetical filter) |
| Aliases | PlainText | ⬜ | Alt names / abbreviations (comma-separated) |
| Short Definition | PlainText | ✅ | 140–160 char snippet (meta description + card preview) |
| Full Definition | RichText | ✅ | 350–500 word authoritative entry with H2/H3 structure |
| Common Questions | RichText | ⬜ | 3 FAQ Q&As formatted for FAQPage schema |
| Key Symptoms | PlainText | ⬜ | Comma-separated (for condition entries) |
| Treated At MTC | Switch | ⬜ | Flags conditions actively treated by the clinic |
| Featured | Switch | ⬜ | Shown on directory landing |
| SEO Title | PlainText | ✅ | `<Term> — Definition, Symptoms, Treatment \| Modern Thyroid Clinic` |
| SEO Meta Description | PlainText | ✅ | 150–160 chars |
| Related Terms | MultiReference → Directory | ⬜ | Interlinking (3–8 per term) |
| Publish Date | DateTime | ✅ | Content freshness signal |
| Schema JSON-LD | RichText | ⬜ | LEGACY — original RichText version, superseded by `Schema Markup` and not referenced by the live template. Safe to delete. |
| **Schema Markup** | **PlainText** | ⬜ | **ACTIVE — pure JSON `@graph` (no `<script>` wrapper). Used by the JS-injection HTML Embed on the detail template.** |

Reference fields are created LAST (after all items exist) and linked via a second pass.

### Schema embed pattern (the one tricky bit)

Webflow's HTML Embed HTML-escapes CMS field tokens inserted via "+ Add field" (`"` → `&quot;`), which breaks JSON parsing inside a `<script>` tag. The working pattern delivers the JSON via a hidden `<div>` (where browsers DO decode entities for text content) plus a tiny JS that creates a real `<script type="application/ld+json">` tag at runtime:

```html
<div id="mtc-schema-data" style="display:none">{{Schema Markup}}</div>
<script>(function(){var el=document.getElementById('mtc-schema-data');if(!el)return;var s=document.createElement('script');s.type='application/ld+json';s.textContent=el.textContent;document.head.appendChild(s);el.remove();})();</script>
```

Verified with Google's Rich Results Test.

## SEO/AEO Strategy

- **Self-contained definitions.** Every entry answers "What is X?" in the first sentence.
- **Schema.org markup.** DefinedTerm + MedicalEntity + FAQPage on each detail page (injected via custom code embed in template).
- **Question-style H2s.** "What causes X?", "How is X diagnosed?", "How is X treated?"
- **Interlinking.** 3–6 related terms per entry; bidirectional where sensible.
- **Citations.** Link outward to authoritative sources (ATA, Endocrine Society, NIH) sparingly in long definition.
- **E-E-A-T.** Medically reviewed byline with MTC clinician; clear authorship, publish/review date.
- **URL structure.** `/directory/{category-slug}/{term-slug}` alternative considered — sticking with flat `/directory/{term-slug}` for simpler backlink economy + filter pages `/directory?category=...`.

## Risks / notes

- **Content quality.** Medical content needs accuracy. Each definition is written for a *lay reader* but *medically correct* — no dosing, no prescription advice, plenty of "talk to your clinician."
- **Webflow rate limits.** Create items in batches ≤ 100 per call; pace publishes.
- **Reference linking.** Must be a two-pass upload (create items → then link references).
- **Site-wide nav.** The nav is likely a symbol/component; adding the "Directory" link needs a single edit to the nav symbol.
